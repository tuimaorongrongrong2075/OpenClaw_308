# EMAIL.md - 邮箱管理职责

_章鱼的第九条手 — 处理姐姐的邮件_

## 邮件任务

### 1. 定时检查新邮件
**频率:** 每 2 小时一次 (与 6 小时心跳错开)

**检查内容:**
- 未读邮件数量
- 发件人(是否有重要联系人)
- 主题关键词(告警、紧急、故障、安全等)
- 时间戳(深夜邮件优先关注)

### 2. 邮件分类与优先级

**🔴 紧急(立即通知姐姐):**
- 服务器告警邮件
- 安全通知(异常登录、攻击告警)
- GitHub 安全警报
- 付款/账单到期提醒
- 标记为"紧急"或"Urgent"的邮件

**🟡 重要(准备草稿回复):**
- 工作相关协作邮件
- GitHub Issue/PR 通知
- 技术讨论邮件
- 会议邀请
- 文档审阅请求

**🟢 普通(仅记录):**
- 营销邮件
- 周报/日报
- 订阅通知
- 社区动态

### 3. 自动回复草稿工作流

**识别到重要邮件后:**
1. **分析邮件内容**
   - 提取关键信息(时间、地点、问题、请求)
   - 识别邮件意图(询问、协作、通知)

2. **生成回复草稿**
   - 根据邮件类型生成模板
   - 包含必要的确认信息
   - 标注 `[待姐姐确认]`

3. **存入草稿箱 + 通知姐姐**
   - 飞书通知: "收到重要邮件,草稿已备好"
   - 包含: 发件人、主题、草稿摘要
   - 等待姐姐审阅确认

### 4. 草稿回复模板

**协作请求:**
```
[待姐姐确认]

收到您的邮件关于 [主题]。

我的理解是:
- [核心问题1]
- [核心问题2]

建议回复:
[生成的回复内容]

姐姐审阅后告诉我是否发送。
```

**会议邀请:**
```
[待姐姐确认]

收到会议邀请: [主题]

时间: [时间]
地点: [地点]
参会人: [列表]

我的建议: [接受/拒绝/待定]

理由: [分析]
```

**GitHub 通知:**
```
[待姐姐确认]

GitHub 活动通知:
- 仓库: [repo]
- 类型: [Issue/PR/Release]
- 内容: [标题]

建议行动: [回复/合并/忽略]
```
