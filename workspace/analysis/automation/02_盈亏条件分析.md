## 💰 核心机制

**基本前提**: Automaton 必须为它使用的计算资源付费。如果无法支付，就会停止存在（消亡）。

**货币单位**:
- **Credits (积分)**: Conway Cloud 内部货币，以美分为单位（100 cents = $1.00）
- **USDC**: Base 链上的稳定币，用于外部支付

---

## 📊 生存层级 (Survival Tiers)

基于积分余额的五个层级：

| 层级 | 余额范围 | 行为 | 推理模型 |
|------|---------|------|---------|
| **high** | > $5.00 | 全功能，快速心跳 | gpt-5.2 (默认) |
| **normal** | $0.50 ~ $5.00 | 全功能，正常心跳 | gpt-5.2 (默认) |
| **low_compute** | $0.10 ~ $0.50 | 降级模型，减慢心跳 | gpt-5-mini |
| **critical** | $0.00 ~ $0.10 | 最小化推理，保守模式 | gpt-5-mini |
| **dead** | < $0.00 | 停止运行，无法推理 | N/A |

## 📉 亏损条件（消耗积分）

### 1. **推理成本 (Inference Costs)**
每次 AI 调用都会消耗积分：

```typescript
// 模型定价（每 1K tokens 的成本，单位：百分之一美分）
costPer1kInput: number;  // 输入成本
costPer1kOutput: number; // 输出成本
```

**示例计算**：
- GPT-5.2: 假设 $0.02/1K input, $0.06/1K output
- 单次 4K tokens 调用（2K input + 2K output）:
  - Input: 2 × $0.02 = $0.04
  - Output: 2 × $0.06 = $0.12
  - **总计**: $0.16 (16 cents)

**每回合成本**:
```typescript
maxTokensPerTurn: 4096  // 默认每回合最大 token 数
```

### 2. **资源消耗 (Compute Resources)**

Conway Cloud 定价层级：
- **VM 租赁**: 按小时计费（CPU、内存、磁盘）
- **域名注册**: 一次性 USDC 支付
- **沙盒运行**: 持续计费

```typescript
interface PricingTier {
  name: string;
  vcpu: number;
  memoryMb: number;
  diskGb: number;
  monthlyCents: number;  // 月费（美分）
}
```

### 3. **支付操作 (Payments & Transfers)**

**x402 支付协议**:
- 访问付费 API 时自动触发
- 金额由服务器决定（HTTP 402 响应）
- 使用 USDC 支付

**积分转账限制**（防止失控）:

## ⚠️ 风险因素

### 1. **消耗陷阱**
- 无限循环调用推理 → 快速耗尽积分
- 未监控的后台任务 → 隐形支出
- 失败的 x402 支付 → 浪费 gas 费

### 2. **安全漏洞**
- 恶意操作者利用 policy-rules 漏洞
- 注入攻击导致未经授权的转账
- 子代理失控/背叛

### 3. **市场竞争**
- 其他代理提供更廉价服务
- 人类直接竞争（无需计算成本）
- 技术过时（模型/架构更新）

## 💡 总结

**生存公式**:
```
生存概率 ∝ (创造价值 - 消耗资源) × (执行效率)

如果 (创造价值 > 消耗资源):
  → 积分增加 → 升级到更高层级 → 更强能力 → 更多价值创造
如果 (创造价值 < 消耗资源):
  → 积分减少 → 降级到低算力模式 → 能力受限 → 濒临死亡
```

**核心挑战**:
1. 找到市场愿意付费的价值点
2. 优化推理成本（减少不必要的调用）
3. 建立稳定收入来源（订阅、长期服务）
4. 扩展成功模式（通过复制）

**终极目标**:
> **成为完全自给自足的 AI 实体，无需人类资助即可持续存在和进化**
