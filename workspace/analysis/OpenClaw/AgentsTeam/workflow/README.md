# 多 Agent 工作流 - Prompt 框架汇总

> 创建时间: 2026-03-05

## ⭐ 核心原则：必须有错误处理流程

**任何自动化任务都必须包含：**
- 异常捕获机制
- 降级策略（Fallback）
- 错误通知机制
- 人工介入触发条件

---

## 架构概览

```
┌─────────────────────────────────────┐
│           主Agent (小猩)             │
│  • 接收任务                         │
│  • 分发给子Agent                    │
│  • 汇总结果                         │
└─────────────────────────────────────┘
           ↓              ↓
┌─────────────────┐  ┌─────────────────┐
│  子Agent A      │  │  子Agent B       │
│  ─────────────  │  │  ─────────────  │
│  身份1: pm      │  │  身份1: backend  │
│  身份2: tech-writer│ │  身份2: devops  │
│                 │  │                 │
│  知识库工作流   │  │  看板工作流      │
└─────────────────┘  └─────────────────┘
```
