# 跨会话上下文同步 - Context Sync

> **创建时间**: 2026-02-28
> **来源**: 飞书群聊讨论

---

## 核心逻辑

```
多轮会话 → 驱动触发 → 优化压缩 → 存储到位 → 按需检索
```

### 驱动何时触发

| 触发方式 | 描述 |
|----------|------|
| 用户指令 | 用户说"记住"/"保存" |
| 自动检测 | 关键结论/代码/配置 |
| 会话结束 | 询问是否需要保存 |

### 优化压缩什么

- ✅ 结论：最终决定、方案选择
- ✅ 代码：关键代码片段
- ✅ 配置：环境变量、参数
- ✅ 待办：下次要做的事
- ❌ 闲聊：日常寒暄（不记录）

### 存储位置

| 位置 | 场景 |
|------|------|
| `memory/YYYY-MM-DD.md` | 当天独立事件 |
| `memory/topics/xxx.md` | 跨天专题 |

### 检索路径

```
用户："接上之前那个..."
       ↓
Step 1: memory/ 快速检索
       ↓
   找到？───未找到？
    ↓          ↓
  返回摘要    Step 2: 查所有 session 历史
                    ↓
              提取 + 压缩 + 确认
                    ↓
              返回精炼版
```

## 2026-02-28 确认更新

**确认内容**：
1. **功能精简**：只做两件事
   - 记录：用户说"记一下" → 存储上下文
   - 检索：用户说"接上之前" → 搜索记忆 → 植入上下文

2. **检索逻辑**：
   - Step 1: 先查 memory/ 文档
   - Step 2: 未找到 → 查所有 session 历史 → 压缩返回

3. **触发方式**：
   - 用户明确说"记一下"或"接上之前"
   - 不自动触发、不猜用户意图

4. **存储位置**：
   - 可指定：`记一下存到 xxx`
   - 默认：`topics/xxx.md`

5. **检索确认**：先返回结果，用户确认后再植入当前会话

1. **问题**：session1 聊话题 A，session2 聊话题 B，想回到话题 A 怎么办
2. **方案**：手动调取历史 / sessions_send / spawn 子 agent / 写共享记忆
3. **结论**：最优是"驱动 → 压缩 → 存储 → 检索"机制
4. **实现**：AGENTS.md 更新 + skill: context-sync
