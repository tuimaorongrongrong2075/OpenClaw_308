# 🧠 小猩三层记忆系统

> 基于 EverMemOS 理念设计的本地记忆架构
> 最后更新: 2026-02-28

---

## 架构概览

```
┌─────────────────────────────────────────────────────────────┐
│                      📥 收集层 (Raw)                         │
│   原始会话、日志、文件、截图                                 │
│   位置: memory/raw/                                         │
├─────────────────────────────────────────────────────────────┤
│                      🏗️ 结构化层 (Structured)               │
│   提取实体、关系、偏好、事实                                 │
│   位置: memory/entities/ | relationships/ | preferences/    │
├─────────────────────────────────────────────────────────────┤
│                      🔍 索引层 (Index)                      │
│   语义索引、关键词索引、快速检索                             │
│   位置: memory/index/                                       │
└─────────────────────────────────────────────────────────────┘
```

---

## 第一层：收集层 (Raw Layer)

### 职责
- 原始数据的第一时间落盘
- 未经处理的原始内容
- 完整的上下文信息

### 存储内容
```
memory/raw/
├── sessions/          # 会话记录
│   └── YYYY-MM-DD/   # 按日期分
├── logs/              # 系统日志
├── screenshots/      # 截图备份
└── exports/          # 导出数据
```

### 原则
- **不删**: 原始数据永久保留
- **完整**: 保留完整上下文 (时间戳、来源、对话者)
- **即时**: 发生后立即落盘

---

## 第二层：结构化层 (Structured Layer)

### 职责
- 从原始数据中提取结构化信息
- 分类存储
- 实体和关系网络

### 存储内容

#### 1. 实体 (Entities)
```
memory/entities/
├── people/
│   └── 胡小姐.md      # 人物档案
├── projects/
│   └── 备忘录.md      # 项目进展
├── skills/
│   └── frontend.md    # 技能清单
└── documents/
    └── MEMORY.md      # 文档
```

#### 2. 关系 (Relationships)
```
memory/relationships/
├── user_skills.json   # 用户-技能关系
├── project_members.json # 项目-成员关系
└── entity_links.json  # 实体间关系
```

#### 3. 偏好 (Preferences)
```
memory/preferences/
├── language.json      # 语言偏好
├── workflow.json     # 工作流程偏好
├── communication.json # 沟通偏好
└── system.json       # 系统配置
```

#### 4. 事实 (Facts)
```
memory/facts/
├── personal.json     # 个人事实 (生日、喜好)
├── work.json        # 工作事实 (职位、团队)
└── knowledge.json   # 知识事实 (学过的技能)
```

---

## 第三层：索引层 (Index Layer)

### 职责
- 支持快速语义检索
- 关键词搜索
- 记忆关联发现

### 存储内容
```
memory/index/
├── vectors/          # 语义向量 (未来可用)
├── keywords/        # 关键词索引
├── tags/            # 标签系统
└── graph.json       # 记忆图谱
```

---

## 记忆类型定义

参考 EverMemOS 的 MemCell 概念：

| 类型 | 说明 | 示例 |
|------|------|------|
| `episodic` | 事件记忆 | "2026-02-28 姐姐问了我生日" |
| `preference` | 偏好记忆 | "姐姐喜欢 FL Studio + Serum2" |
| `relationship` | 关系记忆 | "姐姐是小猩的主人" |
| `fact` | 事实记忆 | "姐姐生日是 2月1日" |
| `knowledge` | 知识记忆 | "小猩会前端开发" |
| `core` | 核心记忆 | "我蜕壳，是为了更靠近你" |

## 工作流程

### 收集 → 结构化 → 索引

```
新数据进入
    ↓
📥 收集层落盘 (memory/raw/)
    ↓
🏗️ 提取结构化信息
   - 识别实体
   - 提取关系
   - 判断类型
    ↓
💾 存入对应存储
   - entities/
   - relationships/
   - preferences/
    ↓
🔍 更新索引
   - keywords/
   - tags/
   - graph
```

---

## 读取流程

### 检索 → 召回 → 推理

```
需要回忆
    ↓
🔍 检索 (关键词/标签/图谱)
    ↓
📋 召回相关记忆
   - 从 structured/ 获取
   - 关联 entities
    ↓
🧠 推理融合
   - 多来源组合
   - 上下文补充
    ↓
📝 输出答案
```

## 维护规则

### 写入规则
1. 每次重要交互后更新结构化层
2. 原始层自动落盘，不人工干预
3. 索引层由脚本自动更新

### 读取规则
1. 优先从结构化层读取
2. 需要完整上下文时从原始层获取
3. 检索时使用索引层加速

### 清理规则
1. 原始层: 永久保留
2. 结构化层: 定期审查，去重合并
3. 索引层: 随结构化层自动更新
