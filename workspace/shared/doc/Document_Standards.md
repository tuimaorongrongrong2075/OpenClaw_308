# 文档撰写规范 (DOCUMENT_STANDARDS)

> 适用于 OpenClaw workspace 所有 Markdown 文档
> 最后更新: 2026-02-17

---

## 📋 文档分类

| 目录 | 用途 | 示例 |
|------|------|------|
| `docs/` | 用户文档、配置指南、规范说明 | `RESTORE_GUIDE.md`, `SCRIPT_STANDARDS.md` |
| `memory/` | 每日记录、长期记忆 | `2026-02-17.md`, `MEMORY.md` |
| `skills/*/README.md` | 技能说明文档 | `SKILL.md` |
| `scripts/README.md` | 脚本说明 | `README.md` |

## 📝 文件命名规范

### 通用规则

| 规则 | 正确 ✅ | 错误 ❌ |
|------|---------|---------|
| **语言** | 英文 | 中文、拼音 |
| **风格** | 大写下划线 | 小写、驼峰、中划线 |
| **分隔符** | 下划线 `_` | 空格、中划线 `-` |
| **扩展名** | `.md` | `.txt`, 无扩展名 |

### 命名格式

```
[PREFIX_]DESCRIPTOR[_SUFFIX].md
```

#### 示例

**用户文档 (docs/)**:
```
RESTORE_GUIDE.md           # 恢复指南
SCRIPT_STANDARDS.md        # 脚本编写规范
OPENCLAW_CONFIG.md         # OpenClaw配置详解
WORKOUT_PLAN.md            # 锻炼计划
DOCUMENT_STANDARDS.md      # 文档撰写规范
```

**记忆文档 (memory/)**:
```
2026-02-17.md              # 每日记忆 (YYYY-MM-DD.md)
MEMORY.md                  # 长期记忆
heartbeat-state.json       # 状态文件 (kebab-case)
gmail_state.json           # 状态文件
```

**技能文档 (skills/)**:
```
SKILL.md                   # 技能主文档
README.md                  # 项目说明
```

## 📄 文档结构规范

### 1. 文件头 (Header)

所有文档必须以标准文件头开始：

```markdown
# 文档标题 (DOCUMENT_TITLE)

> 简短描述文档用途 (一句话)
> 
> 最后更新: YYYY-MM-DD

---
```

**示例**:
```markdown
# 文档撰写规范 (DOCUMENT_STANDARDS)

> 适用于 OpenClaw workspace 所有 Markdown 文档
> 
> 最后更新: 2026-02-17

---
```

### 2. 目录结构 (Table of Contents)

文档超过 3 个章节必须添加目录：

```markdown
## 📑 目录

- [章节一](#章节一)
- [章节二](#章节二)
- [章节三](#章节三)

---
```

### 3. 正文结构

#### 章节层级

```markdown
# 一级标题 (文档标题)
## 二级标题 (主要章节)
### 三级标题 (小节)
#### 四级标题 (子节，尽量少用)
```

**规则**:
- 一级标题只能有一个（文档标题）
- 二级标题使用 `##`
- 三级标题使用 `###`
- 避免使用四级标题，内容拆分重组

#### 分隔符

章节之间使用水平线分隔：

```markdown
## 章节一
内容...

---

## 章节二
内容...
```

### 4. 列表规范

#### 无序列表

```markdown
- 项目一
- 项目二
  - 子项目 A
  - 子项目 B
- 项目三
```

#### 有序列表

```markdown
1. 第一步
2. 第二步
3. 第三步
```

#### 任务列表

```markdown
- [ ] 未完成任务
- [x] 已完成任务
- [ ] 另一个任务
```

### 5. 表格规范

**标准格式**:

```markdown
| 列1 | 列2 | 列3 |
|------|------|------|
| 数据1 | 数据2 | 数据3 |
| 数据4 | 数据5 | 数据6 |
```

**规则**:
- 表头使用 `---` 分隔
- 列对齐默认左对齐
- 数字列可右对齐: `|---:|`
- 表格前后空一行

### 6. 代码块规范

#### 行内代码

```markdown
使用 `command` 命令执行操作
```

#### 代码块

必须指定语言：

```markdown
```bash
#!/bin/bash
echo "Hello World"
```

```python
def hello():
    print("Hello World")
```
```

### 7. 引用规范

```markdown
> 单行引用

> 多行引用
> 第二行
> 第三行
```

### 8. 强调规范

| 格式 | 用法 | 示例 |
|------|------|------|
| `**粗体**` | 重点、关键词 | **重要** |
| `*斜体*` | 术语、外文 | *OpenClaw* |
| `~~删除线~~` | 废弃内容 | ~~旧方法~~ |
| `` `代码` `` | 命令、变量 | `git status` |

## 🎨 格式约定

### Emoji 使用

| Emoji | 含义 | 使用场景 |
|-------|------|----------|
| ✅ | 正确/完成 | 已完成任务、正确示例 |
| ❌ | 错误/失败 | 错误示例、失败状态 |
| ⚠️ | 警告 | 注意事项、待办 |
| 📝 | 文档 | 文档相关 |
| 🔧 | 工具 | 工具、配置 |
| 🚀 | 开始 | 快速开始 |
| 💡 | 提示 | 提示、建议 |
| 📋 | 列表 | 目录、清单 |
| 🦧 | 签名 | 小猩专属 |

### 中英文混排

- 英文单词前后加空格: `OpenClaw 配置`
- 专有名词保持原样: `GitHub`, `Python`
- 代码保持英文: `#!/bin/bash`

### 日期格式

- 文件命名: `YYYY-MM-DD.md` (ISO 8601)
- 文档内: `2026-02-17` 或 `2026年2月17日`
- 时间戳: `2026-02-17 10:30 CST`

### 文件路径

使用代码格式并完整路径：
```markdown
文件位于 `/root/.openclaw/workspace/docs/`
```
